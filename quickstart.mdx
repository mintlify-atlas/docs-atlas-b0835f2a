This guide will walk you through setting up YouTube Manager from installation to uploading your first video.

<Steps>
  <Step title="Install Dependencies">
    Create a virtual environment and install the required packages:

    ```bash
    # Create and activate virtual environment
    python -m venv venv
    source venv/bin/activate  # On Windows: venv\Scripts\activate

    # Install dependencies
    pip install -r requirements.txt
    ```

    <Note>
      This project requires Python 3.8+ and MySQL 5.7+ or 8.0+
    </Note>
  </Step>

  <Step title="Configure Google Cloud Project">
    You need OAuth 2.0 credentials from Google Cloud Console:

    1. Go to [Google Cloud Console](https://console.cloud.google.com/)
    2. Create a new project or select an existing one
    3. Enable the **YouTube Data API v3**
    4. Create OAuth 2.0 credentials (Web application type)
    5. Add authorized redirect URI: `http://localhost:8000/oauth/callback/`
    6. Download the credentials as `client_secret.json`
    7. Place `client_secret.json` in your project root directory

    <Warning>
      Keep your `client_secret.json` file secure and never commit it to version control
    </Warning>
  </Step>

  <Step title="Set Up Environment Variables">
    Create a `.env` file in your project root with the following configuration:

    ```bash
    # Django Settings
    SECRET_KEY=your-django-secret-key-here
    DEBUG=True
    ALLOWED_HOSTS=localhost,127.0.0.1

    # Database Configuration
    DB_NAME=youtube_manager
    DB_USER=root
    DB_PASS=your_mysql_password
    DB_HOST=localhost
    DB_PORT=3306

    # Google OAuth
    GOOGLE_CALLBACK_URL=http://localhost:8000/oauth/callback/
    ```

    <Tip>
      Generate a secure SECRET_KEY using: `python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"`
    </Tip>
  </Step>

  <Step title="Create MySQL Database">
    Create the database for your application:

    ```bash
    # Log into MySQL
    mysql -u root -p
    ```

    ```sql
    CREATE DATABASE youtube_manager CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
    ```

    ```sql
    exit;
    ```
  </Step>

  <Step title="Run Migrations">
    Apply database migrations to set up your schema:

    ```bash
    python manage.py migrate
    ```

    This creates the necessary tables including:
    - `VideoGuardado` - For saved/favorite videos
    - `VideoSubido` - For uploaded videos tracking
  </Step>

  <Step title="Create Superuser (Optional)">
    Create an admin user to access the Django admin panel:

    ```bash
    python manage.py createsuperuser
    ```

    <Info>
      This is optional - users can also authenticate directly via Google OAuth
    </Info>
  </Step>

  <Step title="Start the Development Server">
    Launch your YouTube Manager application:

    ```bash
    python manage.py runserver
    ```

    Your app will be available at `http://localhost:8000`
  </Step>

  <Step title="Authenticate with Google">
    1. Navigate to `http://localhost:8000`
    2. Click the login button
    3. Authenticate with your Google account
    4. Grant the requested YouTube API permissions:
       - `youtube` - Read and manage YouTube account
       - `youtube.upload` - Upload videos
       - `userinfo.email` and `userinfo.profile` - Basic profile info

    <Note>
      The app uses OAuth 2.0 with offline access to maintain your session
    </Note>
  </Step>

  <Step title="Try Your First Action">
    Now you're ready to use YouTube Manager! Try one of these actions:

    **Search for a video:**
    ```python
    # Navigate to /buscar/ and enter a search query
    # The app uses the YouTube Data API v3 search endpoint
    # See core/views.py:109-141
    ```

    **Upload a video:**
    1. Go to `/subir/` (Upload page)
    2. Fill in the video details:
       - Title
       - Description
       - Privacy status (private, public, unlisted)
    3. Select your video file
    4. Click upload

    The video will be uploaded to your YouTube channel and tracked in your database.

    <Tip>
      Videos are stored temporarily during upload and automatically cleaned up afterwards (see `core/views.py:240-254`)
    </Tip>
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="OAuth error: redirect_uri_mismatch">
    Ensure your Google Cloud Console OAuth 2.0 credentials include `http://localhost:8000/oauth/callback/` as an authorized redirect URI.
  </Accordion>

  <Accordion title="Database connection error">
    Verify your MySQL server is running and your `.env` file has the correct database credentials.
  </Accordion>

  <Accordion title="ImportError: No module named 'decouple'">
    Make sure you've installed all requirements: `pip install -r requirements.txt`
  </Accordion>

  <Accordion title="API quota exceeded">
    YouTube Data API has daily quota limits. Check your quota usage in Google Cloud Console.
  </Accordion>
</AccordionGroup>
