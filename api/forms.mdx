## Overview

YouTube Manager uses Django ModelForm to handle video upload data. The `VideoUploadForm` provides validation and widget configuration for uploading videos to YouTube.

## VideoUploadForm

A ModelForm based on the `VideoSubido` model with an additional file upload field.

### Class Definition

```python
from django import forms
from .models import VideoSubido

class VideoUploadForm(forms.ModelForm):
    archivo_video = forms.FileField(
        label="Selecciona tu video", 
        widget=forms.FileInput(attrs={'class': 'form-control', 'accept': 'video/*'})
    )

    class Meta:
        model = VideoSubido
        fields = ['titulo', 'descripcion', 'estado']
        widgets = {
            'titulo': forms.TextInput(attrs={'class': 'form-control'}),
            'descripcion': forms.Textarea(attrs={'class': 'form-control', 'rows': 3}),
            'estado': forms.Select(attrs={'class': 'form-select'}),
        }
```

## Form Fields

### archivo_video

<ParamField path="archivo_video" type="FileField">
  Video file upload field. Not part of the model - used only for handling the upload.
</ParamField>

**Field Configuration:**
- **Type:** `forms.FileField`
- **Label:** `"Selecciona tu video"`
- **Widget:** `forms.FileInput`
- **HTML Attributes:**
  - `class`: `'form-control'` (Bootstrap styling)
  - `accept`: `'video/*'` (Browser file picker shows only video files)
- **Required:** Yes (default)

**Notes:**
- This field is not part of the `VideoSubido` model
- Used to receive the video file before uploading to YouTube
- The file is saved temporarily via `FileSystemStorage` during upload
- Accepts any video format that the browser recognizes as `video/*`

## Validation

### Built-in Validation

The form inherits validation from both Django and the model:

1. **titulo**
   - Required (cannot be blank)
   - Max length: 255 characters

2. **descripcion**
   - Required (cannot be blank)
   - No max length

3. **estado**
   - Required
   - Must be one of: `'private'`, `'public'`, `'unlisted'`

4. **archivo_video**
   - Required
   - Must be a valid file upload

### Custom Validation

No custom validation methods are defined. All validation uses Django's default form validation.

### View Usage

```python
from .forms import VideoUploadForm

@login_required
def subir_video_youtube(request):
    if request.method == 'POST':
        form = VideoUploadForm(request.POST, request.FILES)
        if form.is_valid():
            # Access uploaded file
            video_file = request.FILES['archivo_video']
            
            # Access cleaned data
            titulo = form.cleaned_data['titulo']
            descripcion = form.cleaned_data['descripcion']
            estado = form.cleaned_data['estado']
            
            # Process upload...
            
    else:
        form = VideoUploadForm()
    
    return render(request, 'subir_video.html', {'form': form})
```

### Accessing Form Data After Validation

```python
if form.is_valid():
    # Cleaned and validated data
    titulo = form.cleaned_data['titulo']  # String, max 255 chars
    descripcion = form.cleaned_data['descripcion']  # String, unlimited
    estado = form.cleaned_data['estado']  # One of: 'private', 'public', 'unlisted'
    
    # File upload (from request.FILES, not cleaned_data)
    video_file = request.FILES['archivo_video']  # UploadedFile object
```

## Integration with Views

The form is used in the `subir_video_youtube` view (views.py:182):

1. **GET Request:** Empty form is displayed
   ```python
   form = VideoUploadForm()
   return render(request, 'subir_video.html', {'form': form})
   ```

2. **POST Request:** Form processes uploaded data
   ```python
   form = VideoUploadForm(request.POST, request.FILES)
   if form.is_valid():
       video_file = request.FILES['archivo_video']
       # Save temporarily and upload to YouTube
   ```

3. **After Upload:** Create `VideoSubido` record
   ```python
   VideoSubido.objects.create(
       user=request.user,
       video_id=response['id'],  # From YouTube API
       titulo=form.cleaned_data['titulo'],
       descripcion=form.cleaned_data['descripcion'],
       estado=form.cleaned_data['estado']
   )
   ```

## File Handling Flow

1. User selects video file via `archivo_video` field
2. Form validates file upload on POST
3. View accesses file via `request.FILES['archivo_video']`
4. File saved temporarily using `FileSystemStorage`:
   ```python
   fs = FileSystemStorage()
   filename = fs.save(video_file.name, video_file)
   filepath = fs.path(filename)
   ```
5. File uploaded to YouTube using `MediaFileUpload`
6. Temporary file deleted after upload

<Note>
  The `enctype="multipart/form-data"` attribute is **required** on the `<form>` tag for file uploads to work properly.
</Note>

### Validation Errors
