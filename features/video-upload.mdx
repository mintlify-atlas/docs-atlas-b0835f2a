## Overview

The video upload feature enables users to upload video files directly to their YouTube channel through the application. The system handles file storage, metadata configuration, YouTube API upload, and database tracking.

## Database Model

The `VideoSubido` model (`models.py:20-36`) tracks uploaded videos:

```python
class VideoSubido(models.Model):
    ESTADOS = [
        ('private', 'Privado'),
        ('public', 'Público'),
        ('unlisted', 'No listado'),
    ]

    user = models.ForeignKey(User, on_delete=models.CASCADE)
    video_id = models.CharField(max_length=50, unique=True)
    titulo = models.CharField(max_length=255)
    descripcion = models.TextField()
    estado = models.CharField(max_length=20, choices=ESTADOS, default='private')
    fecha_subida = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return f"{self.titulo} ({self.estado})"
```

## Upload Process

The `subir_video_youtube` view (`views.py:181-259`) implements the complete upload workflow:

```python
@login_required
def subir_video_youtube(request):
    if request.method == 'POST':
        form = VideoUploadForm(request.POST, request.FILES)
        if form.is_valid():
            video_file = request.FILES['archivo_video']
            
            # 1. Guardar archivo temporalmente
            fs = FileSystemStorage()
            filename = fs.save(video_file.name, video_file)
            filepath = fs.path(filename)
            
            # Inicializamos 'media' como None para evitar errores en el finally
            media = None

            try:
                youtube = get_youtube_service(request)
                if not youtube:
                    return redirect('google_login')

                # 2. Configurar metadatos
                body = {
                    'snippet': {
                        'title': form.cleaned_data['titulo'],
                        'description': form.cleaned_data['descripcion'],
                        'tags': ['TubeManager', 'Django'],
                        'categoryId': '22'
                    },
                    'status': {
                        'privacyStatus': form.cleaned_data['estado']
                    }
                }

                # 3. Preparar la subida
                media = MediaFileUpload(filepath, chunksize=-1, resumable=True)
                
                request_upload = youtube.videos().insert(
                    part=','.join(body.keys()),
                    body=body,
                    media_body=media
                )
                
                response = request_upload.execute()
                
                # 4. Guardar referencia en DB
                VideoSubido.objects.create(
                    user=request.user,
                    video_id=response['id'],
                    titulo=form.cleaned_data['titulo'],
                    descripcion=form.cleaned_data['descripcion'],
                    estado=form.cleaned_data['estado']
                )
                
                messages.success(request, f"Video subido con éxito! ID: {response['id']}")
                return redirect('mis_subidos')

            except Exception as e:
                messages.error(request, f"Error al subir: {e}")
            
            finally:
                # --- CORRECCIÓN PARA WINDOWS (WinError 32) ---
                # Cerramos explícitamente el flujo del archivo de la librería de Google
                if media and hasattr(media, 'stream'):
                    try:
                        media.stream().close()
                    except Exception:
                        pass
                
                # Intentamos borrar el archivo. Si falla, lo ignoramos para no romper la app.
                if os.path.exists(filepath):
                    try:
                        os.remove(filepath)
                    except PermissionError:
                        print(f"Advertencia: No se pudo borrar el archivo temporal {filepath} (bloqueado por Windows).")

    else:
        form = VideoUploadForm()

    return render(request, 'subir_video.html', {'form': form})
```

## Upload Workflow

<Steps>
  <Step title="Form Validation">
    Validate form data and extract the uploaded video file from `request.FILES`.
  </Step>
  
  <Step title="Temporary Storage">
    Save the video file temporarily using Django's `FileSystemStorage`.
  </Step>
  
  <Step title="Authenticate">
    Retrieve authenticated YouTube API service from session credentials.
  </Step>
  
  <Step title="Configure Metadata">
    Build the metadata object with title, description, tags, category, and privacy status.
  </Step>
  
  <Step title="Upload to YouTube">
    Use `MediaFileUpload` to upload the video file with resumable upload support.
  </Step>
  
  <Step title="Store Reference">
    Save the YouTube video ID and metadata to the local database.
  </Step>
  
  <Step title="Cleanup">
    Delete the temporary file and close file handles (with Windows-specific error handling).
  </Step>
  
  <Step title="Redirect">
    Show success message and redirect to uploaded videos list.
  </Step>
</Steps>

## File Upload Mechanism

### Temporary Storage

Videos are temporarily stored using Django's FileSystemStorage (`views.py:189-191`):

```python
fs = FileSystemStorage()
filename = fs.save(video_file.name, video_file)
filepath = fs.path(filename)
```

Files are saved to `MEDIA_ROOT` defined in `settings.py:138-139`:

```python
MEDIA_URL = '/media/'
MEDIA_ROOT = os.path.join(BASE_DIR, 'media')
```

### Resumable Upload

The upload uses `MediaFileUpload` with resumable transfer (`views.py:215-221`):

```python
media = MediaFileUpload(filepath, chunksize=-1, resumable=True)

request_upload = youtube.videos().insert(
    part=','.join(body.keys()),
    body=body,
    media_body=media
)

response = request_upload.execute()
```

<Note>
**`chunksize=-1`** uploads the entire file in a single request. For large files, use a specific chunk size (e.g., 1MB = 1024*1024) to enable progress tracking.
</Note>

## File Cleanup

The cleanup logic handles Windows-specific file locking issues (`views.py:240-254`):

```python
finally:
    # --- CORRECCIÓN PARA WINDOWS (WinError 32) ---
    # Cerramos explícitamente el flujo del archivo de la librería de Google
    if media and hasattr(media, 'stream'):
        try:
            media.stream().close()
        except Exception:
            pass
    
    # Intentamos borrar el archivo. Si falla, lo ignoramos para no romper la app.
    if os.path.exists(filepath):
        try:
            os.remove(filepath)
        except PermissionError:
            print(f"Advertencia: No se pudo borrar el archivo temporal {filepath} (bloqueado por Windows).")
```

<Warning>
**Windows WinError 32:** On Windows, file handles may remain open after upload, preventing deletion. The code explicitly closes the media stream before attempting removal.
</Warning>

### Cleanup Steps

1. Close the `MediaFileUpload` stream explicitly
2. Check if the file exists
3. Attempt to delete with `os.remove()`
4. Catch and log `PermissionError` without breaking the application

## Viewing Uploaded Videos

The `mis_subidos` view (`views.py:261-263`) displays user's uploaded videos:

```python
@login_required
def mis_subidos(request):
    videos = VideoSubido.objects.filter(user=request.user).order_by('-fecha_subida')
    return render(request, 'mis_subidos.html', {'videos': videos})
```

Videos are filtered by user and sorted by upload date (newest first).

## Error Handling

Upload errors are caught and displayed to the user:

```python
try:
    # ... upload logic

```

Common errors include:
- Quota exceeded
- Invalid file format
- Network timeout
- Authentication failure
- File size limit exceeded

<Tip>
YouTube's default upload limit is 256 GB or 12 hours, whichever is less.
</Tip>

## Best Practices

- **Default to Private:** New uploads default to private for safety
- **Validate File Size:** Consider adding client-side validation for large files
- **Progress Tracking:** Implement chunked uploads with progress bars for large files
- **Error Recovery:** Store temporary files with unique names to enable retry logic
- **Metadata Defaults:** Pre-populate forms with intelligent defaults (e.g., tags from user's previous uploads)
